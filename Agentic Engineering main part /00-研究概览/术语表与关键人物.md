# 术语表与关键人物

---

## 一、核心术语

### 1.1 基础术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 大语言模型 | Large Language Model (LLM) | 基于海量文本数据训练的大型神经网络模型，能够理解和生成自然语言。代表模型包括 GPT-5、Claude 4、Gemini 2.5 等。 |
| 代理 / 智能体 | Agent | 能够自主感知环境、做出决策并采取行动以实现目标的系统。Simon Willison 的定义："在循环中运行工具以实现目标的事物。" |
| 代理式工程 | Agentic Engineering | 一种专业的软件开发方式，开发者以工程纪律为核心，利用 AI 编码代理来加速和自动化实现工作，同时保持对架构、质量和正确性的全面掌控。由 Karpathy 建议使用，Osmani 于 2026 年 2 月 4 日系统定义。 |
| 氛围编码 | Vibe Coding | 一种极度依赖直觉的 AI 辅助开发方式：开发者不审查代码差异，不阅读生成的代码，仅通过自然语言指令驱动 LLM，盲目接受输出。由 Karpathy 于 2025 年 2 月首次定义。 |
| 代理循环 | Agentic Loop | LLM 代理的核心执行模式：代理在"思考→行动（调用工具）→验证结果→再思考"的循环中反复迭代，直到达成目标或达到停止条件。Willison 于 2025 年 9 月系统性定义。 |
| 编码代理 | Coding Agent | 能够同时生成和执行代码的 AI 工具。代表产品包括 Claude Code（Anthropic，2025.02）、Codex CLI（OpenAI，2025.04）、Cline（Cline Bot Inc，2024下半年）。 |

### 1.2 架构与设计术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 代理-计算机接口 | Agent-Computer Interface (ACI) | 代理与计算机系统交互的方式。类比人机交互（HCI），ACI 关注如何设计工具、文档和接口使 AI 代理能高效使用计算机。Anthropic 首次正式提出。 |
| 上下文工程 | Context Engineering | 围绕 LLM 的上下文窗口进行设计和管理的技术。包括选择性地向代理提供信息、管理对话历史、设计系统提示词、整合检索增强等。 |
| 模型上下文协议 | Model Context Protocol (MCP) | 由 Anthropic 于 2024 年 11 月提出的开放标准协议，为 AI 模型提供与外部工具和数据源的标准化连接方式。类比 AI 领域的"USB 接口"。 |
| 增强型 LLM | Augmented LLM | Anthropic 定义的概念：LLM + 检索、工具、记忆等增强能力，是代理系统的基础构建块。 |
| 提示链 | Prompt Chaining | 将复杂任务分解为一系列步骤，每个 LLM 调用处理上一步的输出，可在中间步骤设置程序化检查点。 |
| 路由 | Routing | 对输入进行分类，并将其导向专门处理该类任务的后续流程。 |
| 并行化 | Parallelization | 让多个 LLM 同时处理一个任务，并对输出进行程序化聚合。包括分片（Sectioning）和投票（Voting）两种变体。 |
| 编排者-工作者 | Orchestrator-Workers | 中心 LLM 动态分解任务并分派给工作者 LLM，再综合其结果的工作流模式。 |
| 评估者-优化者 | Evaluator-Optimizer | 一个 LLM 生成响应，另一个提供评估和反馈，循环迭代直至满足标准。 |

### 1.3 工程实践术语

| 术语 | 英文 | 定义 |
|------|------|------|
| YOLO 模式 | YOLO Mode | 编码代理的一种运行模式，所有工具调用和命令执行都被自动批准，无需人工确认。Willison 警告此模式"极为高效但风险极高"。 |
| 防护栏 | Guardrails | 为代理设置的安全约束，包括输入/输出验证、权限过滤、内容审查等。 |
| 沙箱 | Sandbox | 隔离的执行环境，限制代理能够访问的文件、网络和系统资源。Claude Code 使用 OS 级沙箱（macOS Seatbelt / Linux bubblewrap）。 |
| 检查点 | Checkpoint | 在代理执行过程中的特定状态保存点，允许从失败处恢复而非从头开始。 |
| 测试驱动开发 | Test-Driven Development (TDD) | 先编写测试用例（失败），再编写代码使测试通过，最后重构。Osmani 称其为"Agentic Engineering 与 Vibe Coding 的最大区别"。 |
| AGENTS.md | AGENTS.md | 一种约定俗成的文件格式，放置在项目根目录，为编码代理提供项目上下文。Osmani 2026 年研究显示手工编写优于自动生成。 |
| 人类在环 | Human-in-the-Loop (HITL) | 在代理执行过程中的关键节点引入人类决策。 |
| 子代理 | Subagent | 从主代理委托出来的专门代理实例，负责特定类型的任务。Claude Code 内置 Explore、Plan 等子代理。 |

### 1.4 工具与框架术语

| 术语 | 说明 |
|------|------|
| Claude Code | Anthropic 的终端编码代理，2025 年 2 月发布。功能最完整的编码代理工具。 |
| Codex CLI | OpenAI 的命令行编码代理，2025 年 4 月发布。 |
| Codex Cloud | OpenAI 的云端异步编码代理，支持从手机发起任务。 |
| Gemini CLI | Google 的命令行编码代理，2025 年 6 月发布。特点是超大上下文窗口（1M tokens）。 |
| OpenAI Agents SDK | OpenAI 的轻量级多代理工作流框架，支持 100+ LLM。 |
| Agno | 开源 Python 代理框架 + AgentOS 生产运行时，GitHub 37K+ stars。 |
| LangGraph | LangChain 团队的代理编排框架，使用图状态机作为核心抽象。 |

## 三、关键引语

### 3.1 定义性引语

**Vibe Coding 的原始定义（Karpathy，2025.02）：**

> "There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists... I 'Accept All' always, I don't read the diffs anymore."
>
> — Andrej Karpathy

**Agentic Engineering 的定义（Osmani，2026.02.04）：**

> "Vibe coding = YOLO. Agentic engineering = AI does the implementation, human owns the architecture, quality, and correctness."
>
> — Addy Osmani

**代理循环的定义（Willison，2025.09.30）：**

> "My preferred definition of an LLM agent is something that runs tools in a loop to achieve a goal."
>
> — Simon Willison

**测试的核心地位（Osmani，2026.02.04）：**

> "The single biggest differentiator between agentic engineering and vibe coding is testing."
>
> — Addy Osmani

### 3.2 关于 AI 放大能力

> "AI tools amplify existing expertise. The more skills and experience you have as a software engineer the faster and better the results you can get from working with LLMs and coding agents."
>
> — Simon Willison

> "The irony is that AI-assisted development actually rewards good engineering practices more than traditional coding does."
>
> — Addy Osmani
