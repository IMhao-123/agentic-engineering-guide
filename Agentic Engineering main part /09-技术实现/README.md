# 09-技术实现：Agentic Engineering 实战指南

> **目标：** 教授 Agentic Engineering 的核心实践：测试驱动、严格审查、规划先行
>
> **核心理念：** AI 执行实现，人类掌控架构、质量和正确性
>
> **与 Vibe Coding 的根本区别：** 测试是核心使能条件

---

## 🎯 核心原则（必须理解）

Agentic Engineering ≠ Vibe Coding

| 维度 | Vibe Coding | Agentic Engineering |
|------|-------------|---------------------|
| **核心态度** | 盲目接受所有输出 | 严格审查每一行代码 |
| **测试** | 无或极少 | **测试是核心使能条件** |
| **规划** | 直接开始编码 | 先写规格说明 |
| **审查** | 不审查 | 以人类 PR 标准审查 |
| **产出** | 一次性原型 | 生产级软件 |

> **"测试是 Agentic Engineering 与 Vibe Coding 的最大区别"** — Addy Osmani

## 🚨 关键警告

> **Comprehension Debt 是你最大的敌人**

Anthropic 的研究显示，使用 AI 编码代理的开发者在理解代码上得分 **17% 更低**（50% vs 67%）。

你会看到代码"好像能工作"，但其实：
- 没有人真正理解它是怎么工作的
- 一个小改动导致不相关系统崩溃
- 生产环境出问题时没人知道怎么改

> **本模块最重要的教学：不是如何让代理生成代码，而是如何确保你理解代理生成的代码。**

## 🎓 核心原则 checklist

在学习每个文档时，检查自己是否理解：

- [ ] **测试驱动** — 为什么必须先写测试？
- [ ] **严格审查** — 为什么不能盲目接受代理输出？
- [ ] **规划先行** — 为什么 SPEC 必须包含测试策略？
- [ ] **人类负责** — 谁对代码质量最终负责？
- [ ] **Comprehension** — 你能向同事解释每一行代码吗？
